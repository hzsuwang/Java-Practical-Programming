# 并发编程实战之一：从零搭建多线程测试环境

## 一、为什么需要专门的测试环境？

在Java并发编程中，很多Bug是"概率性"出现的：
- 多线程Bug难以复现：可能运行100次正常，第101次才出问题
- 生产环境不敢随意测试：线上系统不能冒险
- 面试手写代码需要验证：写出来的代码要确保能运行

**一个标准的多线程测试环境能帮你：**
1. 快速验证并发问题的存在
2. 安全地实验各种并发解决方案
3. 为面试和实际项目积累经验

## 三、实战Demo：模拟并发计数器问题

### 场景：10个线程同时给计数器+1000次

**预期结果**：10 × 1000 = 10000  
**实际结果**：几乎总是小于10000

### 代码实现

```java
public class ConcurrentCounterDemo {
    private static int counter = 0;
    
    public static void unsafeIncrement() {
        counter++;  // 这不是原子操作！
    }
    
    public static void runConcurrentTest() throws InterruptedException {
        final int threadCount = 10;
        final int incrementPerThread = 1000;
        
        counter = 0;
        Thread[] threads = new Thread[threadCount];
        
        for (int i = 0; i < threadCount; i++) {
            threads[i] = new Thread(() -> {
                for (int j = 0; j < incrementPerThread; j++) {
                    unsafeIncrement();
                }
            });
            threads[i].start();
        }
        
        for (Thread thread : threads) {
            thread.join();
        }
        
        System.out.println("预期: " + (threadCount * incrementPerThread));
        System.out.println("实际: " + counter);
    }
}
```

## 四、为什么会出现这个问题？

`counter++` 看起来是一行代码，但在JVM层面是三个操作：
1. 读取counter的当前值到寄存器
2. 将寄存器中的值+1
3. 将新值写回counter变量

当两个线程同时执行这三个步骤时，就可能出现：

```
线程A: 读取counter=0
线程B: 读取counter=0
线程A: 计算0+1=1，写回counter=1
线程B: 计算0+1=1，写回counter=1  ← 丢失了一次累加！
```

## 五、解决方案

### 1. 使用synchronized（本课示例）

```java
private static final Object lock = new Object();
private static int safeCounter = 0;

public static void safeIncrement() {
    synchronized (lock) {
        safeCounter++;
    }
}
```

### 2. 使用AtomicInteger（下节课详解）

```java
private static AtomicInteger atomicCounter = new AtomicInteger(0);

public static void atomicIncrement() {
    atomicCounter.incrementAndGet();
}
```

## 六、常见面试题

### Q1: 多线程测试需要注意哪些问题？
- **线程安全**：确保共享资源的正确访问
- **死锁检测**：避免循环等待
- **性能影响**：测试环境要接近生产环境
- **可重复性**：尽量让测试结果稳定

### Q2: 如何模拟高并发场景？
1. 使用线程池控制并发度
2. 使用CountDownLatch同步线程启动
3. 使用CyclicBarrier实现阶段性并发
4. 压力测试工具：JMeter、Gatling

### Q3: 线程安全的测试方法有哪些？
- **单元测试**：JUnit + 多线程
- **集成测试**：模拟真实并发场景
- **压力测试**：长时间高并发运行
- **混沌测试**：随机杀死线程，模拟异常

## 七、总结

1. **环境搭建是第一步**：好的测试环境能节省80%的调试时间
2. **counter++不是线程安全的**：这是Java并发最经典的入门问题
3. **synchronized是最基础的解决方案**：但性能有代价
4. **后续课程将深入更多解决方案**：volatile、Atomic、Lock、并发容器等
